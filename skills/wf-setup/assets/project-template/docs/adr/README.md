# Architecture Decision Records

Use this directory for decisions that are:

1. Hard to reverse
2. Surprising without context
3. The result of a real trade-off

ADRs use sequential filenames:

```text
0001-short-decision-title.md
0002-next-decision-title.md
```

Keep ADRs short. A useful ADR can be one paragraph that states the context, decision, and reason.
